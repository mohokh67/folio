# Folio — Claude Code Instructions

## Commands

```bash
flutter pub get                                        # install deps
flutter run                                            # run on connected device/simulator
flutter test                                           # run all tests
flutter analyze                                        # lint
dart run build_runner build --delete-conflicting-outputs  # regenerate Drift/Riverpod code
```

**Run codegen whenever you change:**
- Any Drift table (`lib/data/tables/*.dart`)
- Any `@DriftAccessor` DAO (`lib/data/database/app_database.dart`)
- Any `@riverpod` / `@Riverpod` annotated provider

## Architecture

```
lib/
  main.dart                        # entry point — init timezone + notifications, then runApp
  app.dart                         # MaterialApp.router, wires themeMode from SettingsRepository
  core/
    providers/                     # Riverpod providers (database, calendar, settings)
    recurrence/                    # frequency enum + recurrence engine (pure logic)
    router/                        # go_router config + onboarding redirect guard
    services/
      occurrence_generator.dart    # generates ExpenseOccurrence rows for a given month
      notification_service.dart    # flutter_local_notifications wrapper
      csv_export_service.dart      # builds CSV + shares via share_plus
    settings/
      settings_repository.dart     # ChangeNotifier — persists to app_settings table
    theme/
      app_theme.dart               # light + dark ThemeData
  data/
    tables/                        # Drift table definitions (source of truth for schema)
    database/
      app_database.dart            # @DriftDatabase, all DAOs, migration strategy
      app_database.g.dart          # GENERATED — do not edit
  features/
    calendar/                      # calendar screen + add-expense form + day sheet
    onboarding/                    # first-run flow (sets currency + default frequency)
    settings/                      # settings screen + categories management
    summary/                       # monthly summary with category breakdown
  shared/widgets/
    scaffold_with_nav.dart         # bottom nav shell (Calendar / Summary / Settings)
test/
  core/recurrence/                 # recurrence engine unit tests
  core/services/                   # occurrence generator integration tests (real in-memory DB)
  core/settings/                   # settings repository tests
  widget_test.dart                 # smoke test
```

## Key Patterns

### Database migrations
`AppDatabase.schemaVersion` is currently **5**. Every schema change requires:
1. Add/modify column in the table file (`lib/data/tables/`)
2. Add `if (from < N)` block in `MigrationStrategy.onUpgrade`
3. Bump `schemaVersion` to N
4. Run `dart run build_runner build --delete-conflicting-outputs`

### Settings
`SettingsRepository` is a `ChangeNotifier` served via `ChangeNotifierProvider`. It loads from the `app_settings` key-value table on startup. All setters (`setCurrency`, `setThemeMode`, etc.) write to the DB and call `notifyListeners()`. The router uses it as `refreshListenable` for the onboarding redirect.

### Occurrence generation
Occurrences are generated lazily — `OccurrenceGeneratorService.generateForMonth()` is called for current ± 1 month when the calendar loads or an expense is created. It is idempotent (checks for existing rows before inserting).

### Notifications
`NotificationService` is initialized in `main()` before `runApp`. `timezone` must also be initialized there (`tz.initializeTimeZones()`). Notification IDs equal the `ExpenseOccurrence.id` so cancellation is O(1).

## Gotchas

- **Never edit `app_database.g.dart`** — it is fully regenerated by build_runner.
- **`timezone` init must happen before `runApp`** — doing it inside a widget causes a crash.
- **Onboarding guard**: the router redirects to `/onboarding` when `settings.onboardingComplete == false`. Until `load()` resolves, `onboardingComplete` is `null` and the guard is a no-op (avoids flash).
- **Calendar dots use normalized `DateTime(y,m,d)`** — always strip time before using a date as a map key.

## Issue Workflow

Before creating a feature branch:
1. `git checkout main && git pull`
2. Create branch: `feat/<issue-number>-<short-description>`

After completing an issue:
1. Never commit to `main`.
2. Update `CLAUDE.md` and `README.md` if behaviour or architecture changed.
3. `gh pr create` targeting `main` — title ≤70 chars, Conventional Commits style.
4. Do not merge the PR — leave for user review.
